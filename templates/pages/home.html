Why Go and HTMX?

I’ve built my own websites using the JavaScript tech stack before, but the constant changes and complexity of the JS ecosystem made me look for something simpler and more stable. I wanted a stack that’s fast, straightforward, and enjoyable to work with.

That’s why I chose Go. Pairing it with HTMX lets me keep things minimal while still building dynamic, responsive web apps without the heavy frontend overhead.

Where is this website hosted?

On my homelab laptop at home. Why? Because it's cheap, fast, and fun!
I'm using Fedora Server to host this website, along with a small reverse proxy to expose it to the internet.
